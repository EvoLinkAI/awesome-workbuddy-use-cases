# 贡献指南

感谢你帮助我们打造最全面的 AI Agent 用例库！

**语言 / Language:** [English](CONTRIBUTING.md) | 简体中文

---

## 如何贡献

### 1. Fork 并克隆仓库

```bash
git clone https://github.com/YOUR_USERNAME/awesome-workbuddy-use-cases.git
cd awesome-workbuddy-use-cases
```

### 2. 找到正确的文件

将你的案例放入最相关的文件：

| 你的案例类型 | 对应文件 |
|---|---|
| 按职业角色分类 | `use-cases/by-profession/` |
| 按任务类型分类 | `use-cases/by-task/` |
| 详细白皮书条目 | `use-cases/whitepaper/` |

同时在 `README.zh-CN.md` 相应板块添加一行摘要。

### 3. 使用模板

#### README / 按任务类型文件中：

```markdown
| 编号 | **[案例名称]** — [一句话场景描述] | `能力标签1` `能力标签2` `能力标签3` |
```

示例：
```markdown
| 101 | **邮件摘要自动推送** — 每天早上将未读邮件按优先级汇总后发送到你手机。 | `IMAP` `NLP 分类` `消息推送` |
```

#### 按职业文件中：

```markdown
- **[案例名称]：** [场景描述。]
```

#### 白皮书表格文件中：

```markdown
| **[编号]. [案例名称]** | [详细场景描述 — 什么触发它，它做什么，输出什么。] | **[核心能力1]**、[能力2]、[能力3]。 |
```

## 案例质量检查清单

提交前，请确认你的案例满足以下标准：

- [ ] **具体且可操作** — 不是"帮我提高效率"，而是"读取 CSV、计算异常值、发送报告邮件"。
- [ ] **明确输入和输出** — 什么文件/数据进入？什么文件/消息/操作产出？
- [ ] **真实工作流** — 基于有人实际定期执行的任务，而非假设性场景。
- [ ] **正确标注能力标签** — 能力标签应反映 Agent 实际需要的能力。
- [ ] **不重复** — 添加前先搜索仓库中是否有类似案例。

## 提交 PR

1. 创建新分支：`git checkout -b add-case-[简短名称]`
2. 进行修改
3. 提交：`git commit -m "添加: [案例名称] 到 [板块]"`
4. 推送并发起 Pull Request

**PR 标题格式：** `添加: [案例名称] — [板块]`

示例：`添加: 租约自动续期提醒 — by-profession/房产中介`

## 能力标签参考

使用统一的标签以便检索：

| 标签 | 含义 |
|---|---|
| `浏览器自动化` | 控制浏览器、填表、网页抓取 |
| `文件读写` | 读写本地文件 |
| `Shell 执行` | 运行终端/命令行命令 |
| `代码执行` | 运行 Python、JS 等脚本 |
| `API 调用` | 调用外部 REST API |
| `OCR` | 从图片/PDF 进行光学字符识别 |
| `多模态` | 处理图片、音频或视频 |
| `NLP` | 自然语言理解/分类 |
| `NLG` | 自然语言生成/文本写作 |
| `定时调度` | 基于时间的触发器、定时任务 |
| `文件监控` | 监听目录变化 |
| `邮件自动化` | 通过 IMAP/SMTP 发送/读取邮件 |
| `日历 API` | 读写日历事件 |
| `向量检索/RAG` | 对本地知识库进行语义搜索 |
| `长期记忆` | 持久化的长期信息召回 |
| `消息推送` | 通过 Telegram、WhatsApp 等发送提醒 |
| `图像处理` | 使用 Pillow 等库处理图片 |
| `Git/Shell` | Git 操作、代码仓库管理 |
| `RPA` | 面向遗留系统的机器人流程自动化 |
| `工作流编排` | 多步骤、多工具的流水线 |
